# <label> 형식
#
# 추가
# Add : 파일(코드나 테스트, 예제, 문서 등)의 추가
# Implement : 기능 구현
# Use : 다른 모듈 사용
# Verify : 검증 코드(단위 테스트) 추가
# Test : 테스트 코드(E2E 테스트) 추가
#
# 변경
# Rename : 파일 이름 변경
# Set : 변수 값 변경
# Move : 코드 위치 변경
#
# 보완
# Revise : 문서 개정
# Update : 코드 보완
# Improve : 코드 향상(호환성, 테스트 커버리지, 성능, 접근성 등)
#
# 수정
# Correct : 코드 수정(문법의 오류나 타입의 변경, 이름 변경 등)
# Fix : 버그 수정
# Refactor : 코드 전면 수정
#
# 기능 변경
# Make : 기존 동작의 변경
# Allow : 기존 허용 범위 제한 변경
# Prevent : 특정한 동작 방지
# Avoid : 특정한 동작 회피 처리
#
# 삭제
# Remove : 코드 삭제
# Delete : 파일 삭제
